SPD Neural Network Layers: A Complete Guide#

This section provides a comprehensive overview of all neural network layers available in SPD Learn. Each layer is designed to work with Symmetric Positive Definite (SPD) matrices while preserving their geometric structure.

Covariance Layer#

Purpose: Transform raw multivariate signals into SPD covariance matrices.

CovLayer#

The CovLayer computes the covariance matrix of input signals:

\[\Sigma = \frac{1}{T-1} (X - \bar{X})(X - \bar{X})^\top\]

where \(X \in \mathbb{R}^{C \times T}\) is the input with \(C\) channels and \(T\) time samples.

Input: (batch, channels, time) → Output: (batch, channels, channels)

import torch
from spd_learn.modules import CovLayer

cov = CovLayer()
signal = torch.randn(32, 22, 500)  # 32 samples, 22 channels, 500 time points
covariance = cov(signal)  # Shape: (32, 22, 22)

Covariance Methods:

  • covariance: Standard covariance (default)

  • sample_covariance: Unbiased sample covariance

  • real_covariance: Real part of complex covariance

  • cross_covariance: Cross-covariance between signals

Bilinear Mapping Layers#

Purpose: Reduce or increase the dimension of SPD matrices while preserving the SPD property.

BiMap#

The BiMap layer performs a bilinear congruence transformation:

\[Y = W^\top X W\]

where \(W \in \mathbb{R}^{n \times m}\) is constrained to the Stiefel manifold (\(W^\top W = I_m\)).

Key Properties:

  • Preserves SPD: If \(X\) is SPD, then \(Y\) is also SPD

  • Orthogonality: The Stiefel constraint prevents information collapse

  • Learnable: \(W\) is optimized during training via Riemannian gradients

Initialization Methods:

  1. kaiming_uniform: Standard Kaiming initialization (default)

  2. orthogonal: PyTorch orthogonal initialization

  3. stiefel: Random matrix on the Stiefel manifold via polar decomposition

import torch
from spd_learn.modules import BiMap

# Reduce from 64x64 to 32x32
bimap = BiMap(in_features=64, out_features=32, init_method="stiefel")

X = torch.randn(16, 64, 64)
X = X @ X.mT + 0.1 * torch.eye(64)  # Make SPD
Y = bimap(X)  # Shape: (16, 32, 32)

Advanced Options:

  • depthwise: Number of parallel bilinear mappings (for depthwise operations)

  • parametrized: Use PyTorch’s orthogonal parametrization (recommended)

  • orthogonal_map: Choose the orthogonal parametrization method

BiMapIncreaseDim#

For dimension expansion (when out_features > in_features), BiMap automatically uses BiMapIncreaseDim internally:

\[Y = P + W X W^\top\]

where \(P\) is an identity padding matrix and \(W\) is a semi-orthogonal projection.

Nonlinearity Layers#

Purpose: Introduce nonlinearity while maintaining the SPD constraint.

ReEig (Rectified Eigenvalues)#

The ReEig layer is the ReLU of SPD networks. It clamps small eigenvalues to a threshold \(\varepsilon\):

\[\text{ReEig}(X) = U \max(\Lambda, \varepsilon I) U^\top\]

where \(X = U \Lambda U^\top\) is the eigendecomposition.

Geometric Interpretation:

  • Prevents matrices from becoming near-singular

  • Ensures numerical stability in subsequent operations

  • Acts as a nonlinear activation in the eigenvalue domain

from spd_learn.modules import ReEig

# Use dtype-aware threshold (recommended)
reeig = ReEig()

# Or specify explicit threshold
reeig_explicit = ReEig(threshold=1e-4)

Threshold Selection:

  • None (default): Uses dtype-aware threshold from numerical configuration

  • 1e-4: Common choice for float32

  • 1e-7: For float64 when more precision is needed

Projection Layers#

Purpose: Map between the SPD manifold and the tangent space (Euclidean).

LogEig (Logarithmic Eigenvalues)#

The LogEig layer projects SPD matrices to the tangent space at the identity:

\[\log(X) = U \log(\Lambda) U^\top\]

This is the key step for trivialization — converting manifold-valued data to vector space where standard classifiers can be applied.

Output Options:

  • upper=True: Extract upper triangular part (symmetric, no redundancy)

  • flatten=True: Flatten to a vector for classification

import torch
from spd_learn.modules import LogEig

# For classification: vectorized output
logeig = LogEig(upper=True, flatten=True)

X = torch.randn(16, 8, 8)
X = X @ X.mT + 0.1 * torch.eye(8)
features = logeig(X)  # Shape: (16, 36) for 8x8 matrices

Dimension Formula: For an \(n \times n\) SPD matrix, the vectorized output has dimension \(\frac{n(n+1)}{2}\).

ExpEig (Exponential Eigenvalues)#

The ExpEig layer is the inverse of LogEig, mapping from tangent space back to the manifold:

\[\exp(X) = U \exp(\Lambda) U^\top\]
import torch
from spd_learn.modules import ExpEig

expeig = ExpEig()
tangent_vector = torch.randn(16, 8, 8)
spd_matrix = expeig(tangent_vector)

Batch Normalization Layers#

Purpose: Normalize batches of SPD matrices while respecting Riemannian geometry.

BrooksBatchNorm#

Implements Riemannian Batch Normalization from Brooks et al. [2019]:

\[\tilde{P}_i = \mathcal{G}^{-\frac{1}{2}} P_i \mathcal{G}^{-\frac{1}{2}}\]

where \(\mathcal{G}\) is the Fréchet mean of the batch.

Effect: Centers the batch around the identity matrix on the manifold.

from spd_learn.modules import BrooksBatchNorm

bn = BrooksBatchNorm(num_features=32, momentum=0.1)

SPDBatchNorm#

Extends BrooksBatchNorm with dispersion normalization from Kobler et al. [2022]:

  1. Centering: Same as BrooksBatchNorm

  2. Dispersion: Normalizes the variance of log-mapped matrices

This is particularly important for domain adaptation in EEG, where different subjects/sessions have varying signal scales.

from spd_learn.modules import SPDBatchNorm

bn = SPDBatchNorm(
    num_features=32,
    momentum=0.1,
    affine=True,  # Learnable scale and bias
)

Parameters:

  • affine: Learnable bias (SPD matrix) and weight (scalar)

  • n_iter: Karcher flow iterations for accurate mean estimation

  • eps: Numerical stability constant

Regularization Layers#

Purpose: Regularize covariance matrices for stability and robustness.

Shrinkage#

Applies Ledoit-Wolf-style shrinkage to covariance matrices:

\[\Sigma_{\text{reg}} = (1 - \alpha) \Sigma + \alpha \cdot \text{tr}(\Sigma) / n \cdot I\]

where \(\alpha \in [0, 1]\) is the shrinkage coefficient (can be learned).

from spd_learn.modules import Shrinkage

shrink = Shrinkage(n_chans=22, init_shrinkage=0.1)

Why Shrinkage?

  • Improves conditioning of ill-conditioned covariance matrices

  • Reduces overfitting, especially with limited samples

  • Critical for small sample sizes (common in EEG)

TraceNorm#

Normalizes matrices to have unit trace:

\[\hat{X} = \frac{X}{\text{tr}(X)}\]
from spd_learn.modules import TraceNorm

tnorm = TraceNorm()

SPDDropout#

Dropout for SPD matrices. During training, randomly sets channels to a scaled identity:

from spd_learn.modules import SPDDropout

dropout = SPDDropout(p=0.2)

Utility Layers#

Vec and Vech#

Vectorization operations for symmetric matrices:

  • Vec: Full matrix vectorization

  • Vech: Upper triangular vectorization (no redundancy)

import torch
from spd_learn.modules import Vec, Vech

vec = Vec()
vech = Vech()

X = torch.randn(16, 8, 8)
full_vector = vec(X)  # Shape: (16, 64)
half_vector = vech(X)  # Shape: (16, 36)

PatchEmbeddingLayer#

Creates patch embeddings from input signals (for transformer-style architectures):

from spd_learn.modules import PatchEmbeddingLayer

# Create patches from 22-channel signals with 10 patches
patch_embed = PatchEmbeddingLayer(n_chans=22, n_patches=10)

Typical Network Architecture#

A standard SPDNet [Huang and Van Gool, 2017] follows this pipeline:

Raw Signal → CovLayer → Shrinkage → BiMap → ReEig → BiMap → ReEig → LogEig → Linear

Example Implementation:

import torch.nn as nn
from spd_learn.modules import CovLayer, Shrinkage, BiMap, ReEig, LogEig


class SimpleSPDNet(nn.Module):
    def __init__(self, n_channels=22, n_classes=4):
        super().__init__()
        self.cov = CovLayer()
        self.shrink = Shrinkage(n_chans=n_channels, init_shrinkage=0.1)
        self.bimap1 = BiMap(n_channels, n_channels // 2)
        self.reeig1 = ReEig()
        self.bimap2 = BiMap(n_channels // 2, n_channels // 4)
        self.reeig2 = ReEig()
        self.logeig = LogEig(upper=True, flatten=True)

        out_dim = (n_channels // 4) * (n_channels // 4 + 1) // 2
        self.classifier = nn.Linear(out_dim, n_classes)

    def forward(self, x):
        x = self.cov(x)
        x = self.shrink(x)
        x = self.bimap1(x)
        x = self.reeig1(x)
        x = self.bimap2(x)
        x = self.reeig2(x)
        x = self.logeig(x)
        return self.classifier(x)

Layer Selection Guide#

When to use which layer:

Scenario

Recommended Layers

Raw EEG → Classification

CovLayer → Shrinkage → BiMap → ReEig → LogEig

Pre-computed covariance

BiMap → ReEig → LogEig (skip CovLayer)

Domain adaptation

Include SPDBatchNorm after BiMap layers

Small samples (< 100)

Use Shrinkage with higher alpha

Large matrices (> 64×64)

Multiple BiMap layers for gradual reduction

Attention mechanisms

Use MAtt model with Log-Euclidean attention
