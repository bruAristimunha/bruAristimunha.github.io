Historical Context: SPD Methods in Neuroscience#

This section provides a guided tour through the evolution of SPD (Symmetric Positive Definite) methods in neuroscience, from classical Riemannian approaches to modern geometric deep learning architectures.

The Origins: Covariance in Signal Processing#

The use of covariance matrices in signal processing has deep roots. For multivariate signals like EEG (electroencephalography), the spatial covariance matrix captures the statistical dependencies between channels:

\[\Sigma = \frac{1}{T} \sum_{t=1}^{T} x_t x_t^\top\]

Early work recognized that covariance matrices are inherently positive definite (assuming non-degenerate signals), leading researchers to explore methods that respect this geometric constraint. Notably, Barachant et al. [2013] introduced a Riemannian-based kernel for classifying covariance matrices with support vector machines, demonstrating that this geometric approach outperforms traditional spatial filtering methods for motor imagery BCI applications.

The Riemannian Revolution (2010-2015)#

The Barachant Framework#

The seminal work of Alexandre Barachant and colleagues [Barachant et al., 2012] introduced Riemannian geometry to the brain-computer interface (BCI) community. Key contributions:

  1. Minimum Distance to Mean (MDM) classifier using geodesic distances

  2. Riemannian mean computation via the Fréchet/Karcher mean

  3. Tangent space projection for compatibility with Euclidean classifiers

# The classic pyRiemann workflow
from pyriemann.estimation import Covariances
from pyriemann.tangentspace import TangentSpace
from sklearn.linear_model import LogisticRegression
from sklearn.pipeline import Pipeline

clf = Pipeline(
    [("cov", Covariances()), ("ts", TangentSpace()), ("lr", LogisticRegression())]
)

Why Riemannian?

Traditional approaches like Common Spatial Patterns (CSP) suffered from:

  • Sensitivity to noise and outliers

  • Difficulty handling non-stationarity

  • No principled way to average covariance matrices

Riemannian geometry provided elegant solutions to all these problems.

pyRiemann: The Foundation#

The pyRiemann library (2015) became the standard tool for Riemannian BCI, providing:

  • Multiple Riemannian metrics (AIRM, Log-Euclidean, Wasserstein)

  • Geometric mean algorithms

  • Classification and regression on manifolds

  • Integration with scikit-learn pipelines

SPD Learn builds upon these foundations, extending them to deep learning.

The Deep Learning Era (2017-2020)#

SPDNet: The First Geometric Deep Network#

Huang and Van Gool [2017] introduced SPDNet, the first deep learning architecture specifically designed for SPD matrices:

Input SPD → BiMap → ReEig → BiMap → ReEig → LogEig → FC → Output

Key Innovations:

  1. BiMap layer: Bilinear transformation preserving SPD structure

  2. ReEig layer: Eigenvalue rectification as nonlinearity

  3. LogEig layer: Projection to tangent space for classification

The architecture draws clear parallels to CNNs:

Riemannian Batch Normalization#

Brooks et al. [2019] addressed the challenge of normalizing batches of SPD matrices. Unlike Euclidean batch normalization (subtracting mean, dividing by std), Riemannian batch normalization:

  1. Computes the Fréchet mean of the batch

  2. Transports each sample to the tangent space at the mean

  3. Centers samples around the identity matrix

This enabled training of deeper SPD networks.

Modern Architectures (2020-Present)#

TensorCSPNet: Multi-Domain Learning#

Ju and Guan [2022] introduced TensorCSPNet, extending SPDNet to learn across temporal, spectral, and spatial domains simultaneously:

EEG → Filter Bank → Multi-scale Covariance → SPDNet per scale → Fusion

Innovations:

  • Learnable filter banks for automatic frequency selection

  • Multi-scale temporal segmentation

  • Cross-domain attention mechanisms

from spd_learn.models import TensorCSPNet

model = TensorCSPNet(
    n_chans=22,
    n_outputs=4,
    n_patches=5,  # Temporal segments
    n_freqs=9,  # Frequency bands
)

TSMNet: Domain Adaptation#

Kobler et al. [2022] addressed the challenge of domain shift in EEG (different subjects, sessions, or devices) through:

  1. SPD Batch Normalization with domain-specific parameters

  2. Dispersion normalization for scale consistency

  3. Unsupervised domain adaptation via batch statistics

from spd_learn.models import TSMNet

model = TSMNet(
    n_chans=22,
    n_outputs=4,
)

This work demonstrated that geometric normalization is key to transfer learning in EEG.

Attention on Manifolds: MAtt#

Pan et al. [2022] introduced manifold attention using Log-Euclidean geometry:

\[\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{\log(Q) \cdot \log(K)^\top}{\sqrt{d}}\right) V\]

This enables transformer-style architectures on SPD manifolds.

GREEN: Wavelets Meet Geometry#

Paillard et al. [2025] combined learnable Gabor wavelets with Riemannian geometry in GREEN (Gabor Riemann EEGNet):

EEG → Gabor Wavelets → Covariance → Shrinkage → SPDNet

The key insight: jointly learning time-frequency representations and geometric features.

SPD Learn: Design and Implementation#

SPD Learn consolidates the methodological advances described above into a cohesive software library. The library addresses several technical challenges that have historically limited the adoption of geometric deep learning methods in neuroscience research.

Core Design Principles:

  1. Native PyTorch Implementation: All operations are implemented in PyTorch without external dependencies, enabling automatic differentiation and GPU acceleration through standard mechanisms.

  2. Trivialization-Based Optimization: Where applicable, the library employs tangent space projections to reduce computational overhead while preserving geometric structure.

  3. Modular Layer Design: SPD-specific layers (BiMap, ReEig, LogEig) can be composed to construct custom architectures beyond the provided reference implementations.

  4. Numerical Robustness: Implementations include dtype-aware eigenvalue thresholds and regularization to ensure stability across single and double precision computations.

Unified Interface:

All models follow a consistent API:

import torch
from spd_learn.models import SPDNet

# All models accept (batch, channels, time) input
model = SPDNet(n_chans=22, n_outputs=4)

# Example input: batch of 16 EEG signals, 22 channels, 500 time points
eeg_signal = torch.randn(16, 22, 500)
output = model(eeg_signal)

Future Directions#

The field continues to evolve:

  1. Foundation Models: Pre-trained SPD networks for EEG

  2. Self-Supervised Learning: Contrastive learning on manifolds

  3. Efficient Architectures: Reduced computational cost

  4. Multi-Modal Fusion: Combining EEG, fMRI, and other modalities

  5. Interpretability: Understanding what geometric networks learn
