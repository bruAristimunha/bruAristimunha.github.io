Deep Learning Integration: Autograd and Trivialization#

Incorporating SPD Geometry into Deep Learning#

Integrating Riemannian geometry into deep learning frameworks like PyTorch presents unique challenges. Standard backpropagation assumes operations in Euclidean space, but SPD matrices live on a curved manifold. This section explains how we can leverage automatic differentiation and trivialization to build differentiable deep learning models on SPD manifolds.

The key insight is that while the manifold is curved, we can use the tangent space at each point as a local Euclidean approximation. By carefully mapping between the manifold and tangent spaces, we can compute gradients and perform optimization.

The Challenge: Gradients on Manifolds#

In standard deep learning, gradients are computed in Euclidean space:

(1)#\[\mathbf{w}_{t+1} = \mathbf{w}_t - \eta \nabla_{\mathbf{w}} \mathcal{L}\]

However, on a Riemannian manifold, we need Riemannian gradients that respect the geometry. The Euclidean gradient must be projected onto the tangent space to obtain the Riemannian gradient:

(2)#\[\text{grad}_{\mathbf{P}} f = \mathcal{P}_{T_{\mathbf{P}}\mathcal{M}}(\nabla_{\mathbf{P}} f)\]

where \(\mathcal{P}_{T_{\mathbf{P}}\mathcal{M}}\) is the projection onto the tangent space at \(\mathbf{P}\).

Automatic Differentiation for SPD Layers#

PyTorch’s autograd can be extended to handle Riemannian operations by defining custom backward() functions. For each manifold operation (e.g., exponential map, logarithmic map), we implement:

  1. Forward pass: Compute the operation on the manifold

  2. Backward pass: Compute the Riemannian gradient using the chain rule and projection

Example: Custom SPD Layer with Autograd#

import torch
import torch.nn as nn
from spd_learn.modules import BiMap


class SPDLayer(nn.Module):
    """
    A simple SPD layer that applies a bilinear transformation.
    Demonstrates automatic differentiation on SPD manifolds.
    """

    def __init__(self, input_dim, output_dim):
        super().__init__()
        # Learnable bilinear transformation matrix
        self.weight = nn.Parameter(torch.randn(output_dim, input_dim))

    def forward(self, X):
        """
        X: Input SPD matrix of shape (..., input_dim, input_dim)
        Returns: Output SPD matrix of shape (..., output_dim, output_dim)
        """
        # Bilinear transformation: W X W^T
        # Automatically differentiable!
        return torch.matmul(torch.matmul(self.weight, X), self.weight.T)


# Example usage
model = SPDLayer(input_dim=10, output_dim=5)

# Input SPD matrix
X = torch.randn(10, 10)
X = X @ X.T + torch.eye(10)  # Ensure SPD

# Forward pass
output = model(X)

# Compute loss (e.g., Riemannian distance to target)
target = torch.randn(5, 5)
target = target @ target.T + torch.eye(5)

from spd_learn.functional import geodesic_distance_spdairm

loss = geodesic_distance_spdairm(output, target)

# Backward pass - gradients computed automatically!
loss.backward()
print(f"Weight gradient norm: {model.weight.grad.norm().item():.4f}")

Trivialization: The Unified Approach#

Trivialization is the process of mapping the manifold to a vector space (and back) in a way that preserves local geometry. This is crucial for deep learning because:

  1. Local Euclidean structure: In a trivialized representation, we can use standard neural network layers

  2. Gradient computation: Gradients are easier to compute in the trivialized space

  3. Computational efficiency: Matrix operations are simpler in vector form

The key maps are:

  • Chart map \(\phi: \mathcal{M} \to \mathbb{R}^d\): Maps manifold points to vectors

  • Inverse chart map \(\phi^{-1}: \mathbb{R}^d \to \mathcal{M}\): Maps vectors back to manifold

Why We Follow the Trivialization Approach#

Our library uses trivialization as a unified framework for several reasons:

  1. Compatibility with PyTorch: Standard layers (Linear, Conv, etc.) expect vector inputs. Trivialization allows us to use these layers seamlessly.

  2. Computational Efficiency: Operating in the tangent space or log-domain often reduces computational cost compared to manifold operations.

  3. Theoretical Soundness: The exponential and logarithmic maps provide diffeomorphisms between the manifold and tangent space, preserving local geometric properties.

  4. Modularity: We can mix Euclidean and Riemannian layers by trivializing at boundaries, creating flexible architectures.

Common Trivialization Strategies#

Log-Euclidean Approach#

Map SPD matrices to the space of symmetric matrices via matrix logarithm:

(3)#\[\phi(\mathbf{P}) = \log(\mathbf{P}) = \mathbf{V} \log(\mathbf{\Lambda}) \mathbf{V}^T\]

where \(\mathbf{P} = \mathbf{V} \mathbf{\Lambda} \mathbf{V}^T\) is the eigendecomposition.

Tangent Space Projection#

Project SPD matrices onto the tangent space at a reference point (often the identity):

(4)#\[\phi_{\mathbf{I}}(\mathbf{P}) = \text{Log}_{\mathbf{I}}(\mathbf{P}) = \log(\mathbf{P})\]

This is equivalent to the Log-Euclidean approach when the reference is the identity [Zanini et al., 2017].

Vector Representation#

Flatten the upper triangular part of the (trivialized) symmetric matrix into a vector:

(5)#\[\text{vec}(\mathbf{S}) = [S_{11}, \sqrt{2}S_{12}, \ldots, \sqrt{2}S_{1n}, S_{22}, \ldots, S_{nn}]^T\]

The \(\sqrt{2}\) factor ensures isometry (distance preservation).

Example: End-to-End SPD Network with Trivialization#

import torch
import torch.nn as nn
from spd_learn.modules import BiMap, ReEig, LogEig


class SPDNet(nn.Module):
    """
    Complete SPD network using trivialization.
    Architecture: SPD -> BiMap -> ReEig -> LogEig -> FC -> Output
    """

    def __init__(self, input_dim, hidden_dim, output_dim, num_classes):
        super().__init__()

        # SPD layers
        self.bimap1 = BiMap(input_dim, hidden_dim)
        self.reeig1 = ReEig()  # Ensure eigenvalues stay positive

        self.bimap2 = BiMap(hidden_dim, output_dim)
        self.reeig2 = ReEig()

        # Trivialization: Map to tangent space and vectorize
        # LogEig computes matrix log and extracts upper triangular part
        self.logeig = LogEig()

        # Standard Euclidean classifier
        # LogEig outputs vectorized upper triangular: output_dim * (output_dim + 1) // 2
        self.fc = nn.Linear(output_dim * (output_dim + 1) // 2, num_classes)

    def forward(self, X):
        # SPD processing
        X = self.bimap1(X)
        X = self.reeig1(X)

        X = self.bimap2(X)
        X = self.reeig2(X)

        # Trivialization: manifold -> vector space
        # LogEig returns vectorized form directly
        X = self.logeig(X)

        # Euclidean classifier
        return self.fc(X)


# Example usage
model = SPDNet(input_dim=32, hidden_dim=16, output_dim=8, num_classes=4)

# Batch of SPD matrices
batch = torch.randn(16, 32, 32)
batch = torch.matmul(batch, batch.transpose(-1, -2)) + torch.eye(32).unsqueeze(0)

# Forward pass
output = model(batch)
print(f"Output shape: {output.shape}")  # Should be (16, 4)

Differences from Other Approaches#

Traditional Riemannian Optimization

Operates entirely on the manifold using retraction and vector transport. More general but computationally intensive.

Euclidean Approximations

Treat SPD matrices as elements of \(\mathbb{R}^{n \times n}\). Simpler but ignores geometric structure and may produce invalid outputs.

Our Trivialization Approach

Balances geometric correctness with computational efficiency. We:

  • Respect manifold structure in critical layers (BiMap, ReEig)

  • Trivialize for efficiency when appropriate (LogEig, vectorization)

  • Leverage PyTorch’s autograd for gradient computation

  • Enable mixing of Riemannian and Euclidean operations

Key Takeaways#

  • Autograd can be extended to Riemannian manifolds via custom backward passes

  • Trivialization maps manifold operations to vector spaces locally

  • Log-Euclidean and tangent space projections are common trivialization strategies

  • Our unified approach balances geometric correctness with computational efficiency

  • The framework enables seamless integration of SPD layers into standard deep learning pipelines
